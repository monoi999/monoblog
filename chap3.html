AI 활용2. Todo App 설계

Todo list GUI App(DB + Tkinter + exe)

📋 Todo 앱 개발 설계 및 진행 단계

개발 프로세스는 크게 환경 설정 → DB 설계 → UI 구현 → 기능 연결 → 배포의 5단계로 나뉩니다.

1. 환경 설정 및 요구사항 정의

  • 기술 스택: Python 3.x, Tkinter (기본 라이브러리), SQLite3 (내장 DB)
  • 주요 기능: 할 일 추가, 목록 조회, 완료 체크(Update), 삭제(Delete)
  • 필요 라이브러리 설치: 실행 파일 제작을 위한 pyinstaller 설치
pip install pyinstaller

2. 데이터베이스(DB) 설계

데이터를 영구적으로 저장하기 위해 SQLite를 사용합니다. 별도의 서버 설치 없이 파일 형태로 관리되어 가볍습니다.

테이블 구조 (tasks)

  • id: 고유 번호 (Integer, Primary Key)
  • content: 할 일 내용 (Text)
  • status: 완료 여부 (Integer, 0: 미완료, 1: 완료)

3. GUI 레이아웃 설계 (Tkinter)

사용자가 마주할 화면을 구성합니다.

  • 입력부: Entry 위젯 (할 일 입력), Button (추가 버튼)
  • 출력부: Listbox 또는 Treeview (목록 표시)
  • 제어부: Button (삭제, 완료 처리)

4. 핵심 기능 로직 구현 (CRUD)

Python 코드와 DB를 연결하는 함수를 작성합니다.

  • Create: 입력창의 텍스트를 DB에 저장하고 목록 갱신
  • Read: 앱 실행 시 DB의 데이터를 불러와 화면에 표시
  • Update: 선택한 항목의 상태를 '완료'로 변경
  • Delete: 선택한 항목을 DB와 화면에서 삭제

5. 실행 파일(EXE) 제작

Python이 설치되지 않은 환경에서도 실행할 수 있도록 패키징합니다.

  • 명령어: pyinstaller --onefile --noconsole main.py
  • --onefile: 모든 라이브러리를 하나의 .exe 파일로 합침
  • --noconsole: 실행 시 검은색 터미널 창이 뜨지 않게 함

6. 단계별 구현 가이드 (요약)

단계 주요 작업 내용 체크포인트
Step 1 sqlite3 모듈로 todo.db 생성 및 테이블 초기화 앱 실행 시 DB 존재 확인
Step 2 Tkinter Tk(), Frame을 활용한 기본 창 구성 위젯 배치(grid/pack) 결정
Step 3 버튼 클릭 시 DB에 데이터를 넣는 함수 작성 예외 처리 (빈 칸 입력 방지)
Step 4 PyInstaller를 이용한 빌드 작업 아이콘(.ico) 설정 여부
