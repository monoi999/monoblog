AI를 활용한 데이터 분석의 이론적 배경

강의 개요 : AI의 기본 개념과 이론적 배경 지식을 이해하고 실무에서 AI를 효과적으로 활용할 수 있도록 기초 지식을 다룬다

❇️ 생성형 AI(Generative AI)

구분 내용
주요 차이점
  • 전통 챗봇(Chatbot): 정보 제공, 정해진 답변만 가능
  • 생성형 AI 챗봇: 학습 데이터로 텍스트, 이미지, 음악, 코드 등 새로운 결과물 생성
  • 생성형 AI 기술(LLM) 접목으로 자연스러운 대화와 창의적 응답 가능
사용 사례
  • 고객 응대 자동화(24시간 문의·불만 접수)
  • 이커머스 지원(상품 추천, 재고 확인, 배송 조회)
  • 기업 서비스, 콘텐츠 제작, 제품 디자인 등 다양한 산업 적용
대표 모델/도구
  • ChatGPT (OpenAI): 텍스트, 음성, 이미지, 비디오를 처리하는 멀티모달 능력(GPT-4o)
  • Claude (Anthropic): 긴 문맥 이해, 추론, 코딩 및 복잡한 알고리즘 설계에 강점
  • Gemini (Google): Google 검색/워크스페이스 연동, 실시간 정보 검색과 데이터 추출에 강점(Gemini 3 Pro)
  • Copilot (GitHub): 개발자의 코드를 실시간으로 완성해 주는 코딩 어시스턴트
  • 그 이외 다양한 분야의 인공지능이 있다. 예를 들어 비디오 및 오디오 생성에 Sora, Suno 등이 있다

❇️ AI 어시스턴트(AI Assistant)와 AI 에이전트(Agent)

👉 AI 어시스턴트는 "지시에 따라 움직이는 비서", AI 에이전트는 "목표를 위해 스스로 생각하는 대리인"

구분 AI 어시스턴트(AI Assistant) AI 에이전트(Agent)
정의
  • 대규모 언어 모델(LLM) 기반, 자연어 처리와 생성형 AI 활용
  • 사용자 입력 이해, 정보 제공, 일정 관리, 이메일 작성 등 능동적 업무 지원
  • 정확한 지시(명령)에 따라 동작하는 비서형 소프트웨어
  • 예: "내일 오전 9시에 알람 맞춰줘"
  • 목표 달성을 위해 스스로 계획·실행·결과 도출
  • 복합적 과업을 자율적으로 처리하는 지능형 시스템
  • API 연동 등 외부 도구 활용, 실제 행동 수행
  • 예: "다음 주 제주도 가족 여행 일정 짜고 숙소 예약해줘"
핵심 구성 요소
  • LLM(거대언어모델)
  • 자연어 처리(NLP)
  • 정확한 명령 수행
  • Brain(추론·계획): 목표를 쪼개고 실행 순서 결정
  • Memory(기억): 과거 이력·선호도 반영
  • Tools(도구/액션): 외부 API·소프트웨어 제어
  • Reflection(자기 성찰): 결과 검토·계획 수정 반복
작동 방식 사용자 지시에 따라 단일 작업을 정확하게 수행 목표 달성을 위해 복합 작업을 스스로 계획·실행·수정
사용 사례 일정 관리, 문서 요약, 컨텐츠 제작, 사내 업무 자동화 등 자율 주행차, 자동 주식 매매 시스템

❇️ 프롬프트 중요성

👉 프롬프트란?

  • 인공지능(AI)이 사용자의 의도를 정확히 파악하여 최적의 결과물을 내놓게 만드는 지시 언어
  • 마치 숙련된 조수에게 명확한 업무 지시서를 주는 것과 같음

👉 프롬프트 엔지니어링(Prompt Engineering)이란?

  • AI 모델(LLM)이 최적의 결과물을 도출하도록 질문이나 명령을 설계하고 최적화하는 기술
잘 작성된 프롬프트는 AI의 잠재력을 극대화하여 생산성을 획기적으로 높여줍니다.

1. 프롬프트의 중요성

  • 응답 품질 결정: 프롬프트의 질이 곧 AI 응답의 질. 모호한 질문은 모호한 답변을 낳음
  • 효율성 및 시간 절약: 명확한 지시는 재질문 횟수를 줄여 업무 시간을 단축
  • 일관성 확보: 구조화된 프롬프트로 원하는 형식의 답변을 지속적으로 얻을 수 있음

2. 프롬프트의 핵심 구조 (4대 요소)

구성 요소 설명 예시
지시 (Instruction) AI가 수행할 구체적인 작업 내용 "논문을 요약해줘"
문맥 (Context) 배경 정보, 역할(페르소나), 목적 제공 "너는 10년 차 IT 전문 기자야"
입력 데이터 (Input) AI가 참고하거나 분석해야 할 실제 자료 [논문 텍스트 또는 PDF 내용]
출력 지시 (Format) 답변의 형식, 길이, 톤앤매너 지정 "3줄 요약, 개조식으로 작성해줘"

3. 프롬프트 예시 (Before & After)

일반적인 질문 (Before) 구조화된 프롬프트 (After)
"마케팅 전략 좀 짜줘."
  • [Persona] 너는 스타트업 전문 마케팅 컨설턴트야.
  • [Context] 이제 막 출시된 친환경 텀블러의 초기 인지도를 높이려 해. 타겟은 2030 직장인이야.
  • [Task] 인스타그램에서 실행할 수 있는 바이럴 이벤트 아이디어 3가지를 제안해줘.
  • [Format] 각 아이디어는 '이벤트명, 실행 방법, 기대 효과' 순으로 정리해줘.

4. 실습 가이드

  • 직접 AI에게 아래 문장을 복사하여 입력해 보며 차이를 느껴보세요.
  • 기초 실습: "양자역학에 대해 설명해줘." (결과 확인 후 다음 단계로 진행)
  • 구조화 실습: "너는 초등학교 선생님이야. 이제 막 과학을 배우기 시작한 10살 아이가 이해할 수 있게 양자역학을 아주 쉬운 비유를 들어 설명해줘. 마지막엔 한 문장 요약을 덧붙여줘."

5. 데이터 처리·분석을 위한 프롬프트 가이드

  1. 데이터 전처리 및 정리 (Cleaning)
    • 프롬프트 예시:
      너는 데이터 엔지니어야. 아래 설명하는 데이터셋의 전처리 코드를 Python Pandas로 작성해줘.
      데이터 상황: income 열에 결측치가 있고, age 열에는 150세 이상의 이상치가 포함되어 있어. user_id는 중복된 행이 있을 수 있어.
      요청 사항: 중복 행을 제거하고, income의 결측치는 중앙값으로 채워줘. age가 100세를 초과하는 데이터는 이상치로 판단해 제거하는 코드를 짜줘. 각 단계마다 주석을 달아줘.
  2. 탐색적 데이터 분석(EDA) 및 시각화
    • 프롬프트 예시:
      너는 데이터 분석가야. 첨부한 CSV 데이터셋을 분석하기 위한 시각화 전략을 세워줘.
      분석 목표: 분기별 매출 트렌드와 가장 많이 팔린 제품 TOP 5를 알고 싶어.
      요청 사항: 어떤 차트(막대, 선 그래프 등)가 적절한지 이유와 함께 제안하고, Python(Seaborn 라이브러리)으로 시각화 코드를 작성해줘. 배경색은 깔끔한 흰색으로 설정해줘.
  3. 비즈니스 인사이트 및 가설 도출
    • 프롬프트 예시:
      너는 30년 차 시니어 전략 컨설턴트야. 아래 분석 결과를 바탕으로 비즈니스 인사이트를 도출해줘.
      분석 결과: 주말 매출이 평일보다 40% 높지만, 방문 고객 수는 평일이 더 많음. 20대 고객의 이탈률이 최근 3개월간 15% 증가함.
      요청 사항: 이 현상의 원인을 3가지 가설로 세워보고, 이탈률을 낮추기 위해 즉시 실행 가능한 마케팅 액션 플랜을 제안해줘.

❇️ [워크샵] 타이타닉 데이터셋을 이용한 전처리 및 분석

1️⃣ 데이터셋 정보 파악하기

- 'data/titanic.csv' 파일 읽어와 다음 음 코드를 실행하여 확인한다 ```python import pandas as pd df = pd.read_csv('data/titanic.csv') df.head() df.info() ``` print("Hello from Python in HTML!") result = 1 + 2 print(f"1 + 2 = {result}")
  1. 데이터 탐색 및 전처리 (Cleaning)
    • 데이터의 생김새를 파악하고 결측치(Missing Value)를 처리하는 단계
    • 프롬프트 예시:
      너는 데이터 분석 전문가야. Python의 seaborn 라이브러리에 내장된 타이타닉 데이터셋을 불러와서 전처리를 수행하는 코드를 작성해줘.
      작업: 각 컬럼의 결측치 개수를 확인하고, 'age'는 중앙값으로, 'embarked'는 최빈값으로 채워줘. 'deck' 컬럼은 결측치가 너무 많으니 삭제해줘.
      출력: 단계별 설명과 함께 실행 가능한 코드를 제공해줘.
  2. 탐색적 데이터 분석 (EDA)
    • 생존율에 영향을 미친 핵심 요인을 시각화로 찾아내는 단계
    • 프롬프트 예시:
      타이타닉 데이터에서 객실 등급(pclass)과 성별(sex)에 따른 생존율(survived) 차이를 분석하고 싶어.
      시각화: seaborn을 사용해 성별과 객실 등급별 생존자 수를 보여주는 막대 그래프(countplot)를 그려줘.
      인사이트: 그래프를 통해 알 수 있는 주요 특징(예: '여성과 아이 먼저' 원칙이 지켜졌는지 등)을 3줄로 요약해줘.
  3. 가설 검증 및 인사이트 도출
    • 단순 수치를 넘어 비즈니스나 정책적 시사점을 도출하는 단계
    • 프롬프트 예시:
      너는 역사적 데이터 통계 학자야. 타이타닉 생존 데이터 분석 결과 '1등석 여성의 생존율이 가장 높았다'는 점이 확인됐어.
      요구사항: 이 결과가 당시의 사회적 배경과 어떤 연관이 있는지 분석하고, 만약 현대의 재난 대응 시스템에 이 데이터를 적용한다면 어떤 구조 우선순위 전략을 세울 수 있을지 제안해줘.
실무와 취업 준비를 위한 팁:
- AI의 기본 원리와 데이터의 중요성을 이해하면 실무 적응력이 높아집니다.
- 다양한 AI 활용 사례를 찾아보고, 데이터 분석 프로젝트 경험을 쌓는 것이 중요합니다.
