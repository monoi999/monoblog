# 모노의 강의 자료

## AI 관련 기본 지식

### 챗봇(Chatbot)과 생성형 AI(Generative AI)
- 기존 데이터 내에서의 응답(챗봇)'인가, '학습 데이터를 바탕으로 새로운 콘텐츠 생성(생성형 AI)'인가 하는 점이다
- 전통적인 챗봇은 정보 제공이나 정해진 답변만 가능한 형태라면 생성형 AI기반 챗봇은 기업의 방대한 내부 데이터를 학습하여 사용자 질문에 대한 맞춤형 정보를 생성하고 요약하여 제공한다
- AI 챗봇의 사용 사례로는 고객 응대 자동화로 단순 문의·불만 접수를 처리해 24시간 고객센터 역할 수행, 
이커머스 지원으로 상품 추천, 재고 확인, 배송 조회 등 쇼핑 편의 제공등이 있다

### 생성형 AI(Generative AI)
- 생성형 AI(Generative AI)는 사용자의 프롬프트(명령어)를 기반으로 새로운 콘텐츠를 생성하는 인공 지능 기술입니다. 기존 AI가 데이터 분류나 예측에 집중했다면, 생성형 AI는 학습된 데이터의 패턴을 활용하여 독창적인 결과물을 생성한다
- 대화, 글쓰기, 코드 생성에 GPT-4, Claude 3등 기술을 이용하며 이미지 생성에  DALL-E 3, Midjourney 등 기술을 오디오 및 비디오 생성에 Sora등의 기술을 이용한다

### AI 어시스턴트(AI Assistant)
- 대규모 언어 모델(LLM) 기반 AI 어시스턴트는 자연어 처리(NLP)를 사용하여 자연어 처리(NLP)와 생성형 AI를 기반으로 사용자 입력 이해, 정보 제공, 일정 관리, 이메일 작성 등 능동적인 업무 처리를 지원하는 지능형 소프트웨어입니다
- OpenAI의 ChatGPT, Google의 Gemini, Anthropic의 Claude 등이 대표적으로 음성 명령이나 텍스트를 통해 멀티태스킹을 지원하며, 생산성을 높여주는 역할을 합니다. 
- 생성형 AI는 무언가를 '생성'하는 능력이 핵심이며 AI 어시스턴트는 사용자와 대화하며 비서처럼 문제를 해결해 주는 '인터페이스'와 '기능'이 핵심이다
- 즉 생성형 AI는 '기술적 엔진'(두뇌 역할)이고 AI 어시스턴트는 생성형 AI를 탑재한 '제품/서비스 형태'(도구)입니다
- 단순 질문 답변(챗봇)을 넘어 스스로 판단하여 업무를 수행하는 AI 에이전트로 발전하고 있습니다
- 주요 개발 기업 및 서비스
    - 구글(Google): Gemini
    - 오픈AI(OpenAI): ChatGPT
    -  Anthropic : Claude
    - 마이크로소프트(Microsoft): Copilot
    - 아마존(Amazon): Amazon Q
    - 기타: 삼성전자(갤럭시 AI), 어도비(Adobe Campaign)

### AI 에이전트(Agent)
- 사람이 설정한 목표(Goal)를 달성하기 위해 스스로 계획을 세우고 실행하며 결과를 도출하는 자율형 지능 시스템이다
- 예를 들어  "최신 AI 트렌드 10가지를 조사해서 PPT로 만들고 팀원들에게 이메일로 공유해줘"라는 명령 하나로 자료 수집, 문서 작성, 발송까지 완료할 수 있다
- AI 에이전트의 4가지 핵심 구성 요소
    - 브레인 (추론 및 계획): LLM(거대언어모델)을 두뇌로 사용하여 복잡한 목표를 작은 단위의 작업으로 쪼개고 실행 순서를 결정합니다.
    - 기억 (메모리): 과거의 작업 이력이나 사용자의 선호도를 기억하여 맥락에 맞는 최적의 결정을 내립니다.
    - 도구 사용 (액션): 인터넷 검색, 이메일 발송, 결제 시스템, 엑셀 조작 등 외부 API나 소프트웨어를 직접 제어합니다.
    - 자기 성찰 (Reflection): 실행 결과를 스스로 검토하고, 오류가 있다면 계획을 수정하여 목표를 달성할 때까지 반복합니다

- AI 에이전트 종류
    - Google Project Jarvis: Gemini 2.0 모델을 기반으로 크롬 브라우저 내에서 쇼핑, 항공권 비교, 정보 수집 등을 자율적으로 수행하도록 설계된 도구
    - Microsoft Copilot Agents: Excel, Teams, Outlook 등 M365 앱 간을 오가며 데이터를 분석하고 이메일을 발송하는 등 업무 협업에 최적화된 에이전트
    - Claude Code: Anthropic에서 출시한 자율형 코딩 에이전트로, 프로젝트 전체의 맥락을 이해하고 버그 수정 및 성능 최적화(테스트와 배포)를 주도적으로 수행합니다
    - OpenAI Operator: 웹 브라우저를 직접 제어하여 항공권 예약, 티켓 구매, 식당 예약 등 복합적인 과업을 스스로 수행합니다.



- 이 기술은 업무 효율을 높이고 창의적인 작업을 돕지만, 할루시네이션(환각 현상)으로 인한 잘못된 정보 제공이나 저작권 및 보안 문제와 같은 한계점도 있습니다
- 유용한 AI 도구들을 분야별로 정리

📘 AI 기반 데이터 처리 및 분석 실무 과정
- 1부: 데이터 분석의 기반과 시각화 (기초)
    - 데이터 핸들링의 정석: Pandas와 NumPy를 활용한 대용량 데이터 전처리
    - 정적인 그래프를 넘어선 시각화: Matplotlib을 넘어 Plotly와 Seaborn을 이용한 인터랙티브 차트 생성
    - 실습: 반도체 웨이퍼 맵(WM-811K) 데이터셋 탐색 및 결함 패턴 시각화
- 2부: 머신러닝 & 딥러닝 핵심 알고리즘 (핵심)
    - 단순 암기가 아닌, 전자공학의 수학적 배경(선형대수, 통계)을 AI 모델링에 연결합니다.
    - 회귀와 분류: 센서 데이터를 이용한 수치 예측 및 이상치 탐지
    - CNN을 활용한 패턴 인식: 생산 라인 내 불량품 판정을 위한 이미지 분류
    - 시계열 데이터 처리: RNN/LSTM을 활용한 장비 고전 예지 보전(PdM)
    - 실습: 웨이퍼 결함 유형(Donut, Scratch 등) 자동 분류 모델 구현
- 3부: AI 웹 대시보드 제작 (결과물 구현)
    - 분석 결과를 단순한 '보고서'가 아닌 '동작하는 서비스'로 만드는 단계입니다
    - Streamlit & Dash 기초: HTML/CSS 없이 파이썬만으로 인터랙티브 웹 앱 만들기
    - UI/UX 디자인: 사용자 친화적인 필터, 사이드바, 실시간 업데이트 기능 구현
    - API 연동: 모델을 웹 서비스와 연결하여 실시간 추론 결과 보여주기
    - 실습: "나만의 반도체 공정 모니터링 대시보드" 제작
- 4부: 프로젝트 고도화 및 포트폴리오 (심화)
- 취업 시장에서 차별화될 수 있도록 '자동화'와 '배포' 개념을 추가합니다
- 워크플로우 자동화: make 툴이나 스케줄러를 활용한 데이터 파이프라인 자동화
- 금융 데이터 응용: 주식/ETF 데이터를 수집하여 자신만의 투자 지표 대시보드 만들기
- 취업 전략: GitHub을 활용한 코드 관리와 AI 기술 블로그 작성법

💡 강사님을 위한 강의 팁
- 도구의 현대화: 전통적인 GUI 방식(Tkinter 등)보다는 현업에서 데이터 대시보드로 가장 많이 쓰이는 Streamlit을 주력으로 가르치시는 것을 추천합니다. 결과물이 웹으로 바로 나오기 때문에 학생들의 성취도가 매우 높습니다.
- 데이터의 다양성: 제조 데이터(반도체)로 시작해 관심도가 높은 금융(ETF), 미디어(Spotify) 데이터 순으로 확장하면 지루함을 줄일 수 있습니다.
